*** an environment variable
ConnectionStrings__Funnelica

the double underscore (__) — it represents : in configuration paths.

*** To make it permanent:
/> setx ConnectionStrings__Funnelica "Data Source=_the_server_;Initial Catalog=_the_db_;Integrated Security=True;Trust Server Certificate=True;"
for instance: Data Source=(local)\SQLEXPRESS

Note: You need to open a new command prompt for the changes to take effect.

*** To check if it was set correctly:
/> echo %ConnectionStrings__Funnelica%

or
/> set | find "ConnectionStrings"

In web.config as environment variable it will be represented as:
	<environmentVariable name="ConnectionStrings:Funnelica" value="Data Source=(local)\SQLEXPRESS;... />.

IIS localy
	
IIS creates a virtual account: IIS AppPool\Funnelica
Application Pool settings:	
	.NET CLR Version: No Managed Code
	Managed Pipeline Mode: Integrated
	Identity: ApplicationPoolIdentity

Connection string localy can be set as an environment variable in IIS, via site Configuration Editor.
	
SQL Server Configuration
1) Create a new database server login in Security/Logins : IIS AppPool\Funnelica
2) Create a new Funnelica DB user in Security/Users : user name- IIS AppPool\Funnelica | login name- IIS AppPool\Funnelica
